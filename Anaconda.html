Background

	Anaconda = langugage, python + R for scientific computing
	Used for creating separate environments with different files/packages/dependencies 
	Useful when: 
		1. sharing code with people, others have diff package managers + different package versions and dependencies 
		2. different projects need different versions of python, which can be kept separate in environments

Download

	1. Download .sh file: https://www.anaconda.com/download/
	2. bash <conda-installer-name>-latest-Linux-x86_64.sh
	3. follow prompts to download, accept default configs 
		3.5 Remove conda: rm -rf ~/path/to/conda 
				- conda init --reverse --all
				- rm -rf ~/.condarc ~/.conda ~/.continuum
	4. eval "$(/home/tina/anaconda3/bin/conda shell.YOUR_SHELL_NAME hook)" 
			- use "bash" for YOUR_SHELL_NAME
			- find shell name: ps -p $$ 
			- needed so conda will be recognized in current shell (temporary)
	5. Instead of 4: 
			- Enter conda with #4
				- terminal should look like: (base) tina@debian:~$ 
			- conda config --set auto_activate_base true 
			   - activates conda on startup
			- conda init 
			  	- install's conda's shell functions 
				- edits /home/tina/.bashrc 
			- open new shell
				- should automatically see (base) in terminal 
				- <conda> to check if running 
	6. check conda configurations 
				conda config --show

Getting Started

Creating Environments 

	1. Creating new environment 
			conda create -n <env-name>  
				- want to create new environment for any new project or workflow 
	2. OR add packages while creating environment 
			conda create -n <myenvironemnt> python numpy pandas
	3. default environment location 
			/home/tina/anaconda3/envs/<myenvironment>
	4. list all environments and locations (active env has *)
			conda info --envs 
	5. activate environment (do before installing packages) 
			conda activate <myenvironment>
	6. change current environment back to default "base"
			conda activate 
	7. deactivate environment (do before deleting, or just to to stop working in it)
			conda deactivate 
	8. conda remove --name <myenvironment> --all
	
	For AI project:
		1. conda create -n ai-project python numpy
		2. conda activate ai-project
		3. install EVERYTHING for project in <ai-project>!!

Installing Packages 

	1. activate environment and install packages 
			conda activate <myenvironment>
			conda install matplotlib 
	2. specify environment and install 
			conda install --name <myenvironment> matplotlib
	3. list installed packages 
			conda list

Specifying Channels 

	Channels = location on computer or someone on internet where packages stored. Conda looks for packages in default channels. 
	
	1. install package from another channel 
			conda install conda-forge::numpy
				- install numpy package from conda-forge channel

Updating conda 

	1. check version 	
			conda --version
	2. update
			conda update conda

Pip vs Conda

	- conda = system package manager, pip = python package manager
	- use conda to install all needed packages from conda channels then use version of pip from conda to install pip dependencies into environment

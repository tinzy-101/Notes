Deleting Elements in BST

  1. ask: does the node you want to delete have a left child?
    • if yes:
      • look at the rightmost element in the left sub tree, fetch content, and overwrite curr node with that value
      • when moving the rightmost element up, if there is left sub tree, it should be adopted by the parent as left child
    • if no:
      • give up right subtree to be adopted by ur parent (the right sub's grandparent node)
  2. if duplicate elements in bst:
    • if trying to insert a duplicate values of one of the nodes, go to the left sub tree of that node
  3. deleting with duplicates
    • need to figure out which you want to delete or if you want to delete all copies
  4. iterators
    • need a uniform way to traversing the elements in the tree
    • have an iterator class
    • tree.begin() = the root element
    • tree.end() = nullptr (moved beyond the bst)
    • ++iterator = can move the iterator like an index (preinc overloaded for the class)
    • want to get iterator object out of bst tree:
      • visit all nodes in bst (e.g. in order)
      • write the values in the nodes into a vector (NOT the node ptrs)
      • traverse the vector of the sorted elemetns, and store the "current" as an index in the vector (current is an int)
    • an iterator object only stores current + vector of all elements
  5. define begin() and end()
    • in the BST tree, NOT in the iterator class
  6. path function
    • given a node in bst tree, it will give the order that you traverse nodes to reach the desired node (stored in vector)
  7. colon means inheritance

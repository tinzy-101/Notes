Templates

  1. if a function passes in datatype "typedef" by value, can't use for multiple
  2. template functions
    • underlying data type given a name, but not pinned down to a specific type in the function implementation
    • compiler will determine the data type to give you the right version of the function (invocation time)
  3. 2 methods
    • using typedef
      • define typedef int item, pass in by value "item a" and "item b", do a comparison and return largest
      • return type of function also item
      • once you use the function once on ints, it will only ever work on ints
    • using template
      • define template prefix, use anonymous class type: template <class Item>
      • define funciton header: Item maximal(Item a, Item b)
      • this type is determined at compile time (will return the right function to use depending on the type)
      • can use on any types that operators in the funcitons are defined for
  4. Item temp =x
    • copy constructor invoked
  5. cannot pass in long as the size of an array, because its too big, so it cant do pointer arithmetic
    • usually pass in size_t n (unsigned int) --> only nonneg ints)
    • DON'T do: const size_t SIZE = 5; unless you are using a template in the function header so that you can
      • instantiate the function with passing in the right type for the size of the array
  6. e.g. template class to insert into a sorted list
    • if you pass in by value, e.g. Item entry, will need a copy constructor
  7. bag class using template
    • want to define bag class using template, instead of using only a typedef
  8. when using template class
    • NEED implementation and function prototypes in the header file (cant split into implementation file)
    • OR can put implementation into a .template file (its just a text file) --> will lose color syntax
      • #include "bag4.template" in the header file after the class definition
  9. once you leave the class, anything with a bag type that you define needs to be instantiated
    • e.g. const typename bag<Item>::size_type bag<Item>::DEFAULT_CAPACITY
  10. every function becomes a template function
    • need to have a template header preceding each function def
    • even nonmember function, who use the class, need to use the template
    • EXCEPT constructor, that you dont need to instantitate
  11. if want to use a return type defined in the class
    • bag<Item>::size_type --> need to add "typename"
    • only need for return types, if using size_type in the body of a function compiler knows that its the name of a type
  12. use Item instead of valuetype
  13. if you have default values in function: specify them once in header files
  14. dont add any directives, using namespace std
    • need to manually: std::cout

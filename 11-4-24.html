AVL Tree Intro

  1. if you dont have a well balanced tree, then a BST turns a search into a O(n) operation
  2. avl guarantee: height difference between left and right subtree at most +-1
    • dont need a perfectly balanced bst to get O(logn) search
  3. balance factor: height of right sub tree - height of right subtree
    • balance factor of a parent node must be >=1, else need to some child node so that tree is more balanced
  4. different cases of imbalance:
    • RR rotation: right tree is too long
      • when right heavy: balance factor = 1
      • parent's balance factor +2, childs balance factor 0 or +1 (match sign)
      • do a left rotation
    • LL rotation: lef tree is too long
      • when left heavy: balance factor = -1
      • parent's balance facotr -2, childs balance factor 0 or -1 (match sign)
      • do a right rotation
    • LR rotation: left tree is too long
      • parent's balane facotr -2, childs balance factor +1 (diff sign)
      • do a left rotation then a right rotation
    • RL rotation: right tree too long
      • parent's balance factor +2, childs balance factor -1 (diff sign)
      • do a right rotation then a left rotation
  5. designing AVL tree
    • is an inherited class of BST tree, each node had content, left ptr, right ptr, but avl tree also has "height" property

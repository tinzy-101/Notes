32.1 Introduction

	--> java has built in support for multithreading

32.2 Thread Concepts

	--> thread = mech for running a task + launch multiple threads from program concurrently
	--> single processor: threads shared CPU time (OS needs to schedule)
	--> multithreading: good for making program more interactive + better performance 
	--> Java: each task = instance of "runnable object"
		--> thread = object that facilitates exec of task

32.3 Creating Tasks and Threads

	--> tasks = objects, so need to define a class for tasks
	--> class implements "runnable" interface 
		--> "runnable" interface only has "run()" method
		--> need to implement run() method to tell sys how thread runs
	
	//custom task class
	public class TaskClass implements Runnable {
		public TaskClass(...) {
			...
		}
		
		//implement run method in Runnable
		public void run() {
			//how to run custom thread
		}
	}
	
	//client class
	public class Client() {
		public void someMethod() {
			//create instance of task class
			TaskClass task = new TaskClass(...);
			
			//create thread
			Thread thread = new Thread(task);
			
			//start thread
			thread.start();
		}
	}
	
	--> JVM executes task by invoking task's run() method

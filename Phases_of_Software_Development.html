Preconditions and Postconditions:

  • procedural abstraction = don't need to know how a function works, just know what it does
  • precondition = condition required for function to be called
  • postcondition = condition that must be held after a function is called (what function should do)
  • "assert" keyword = used to check precondition is fulfilled
    • #insert <cassert>
    • assert(c >= MINIMUM_CELSIUS)
      • if true, continue with program
      • if false, stop the program and print error message

Testing and Debugging

  • test inputs should include boundary values
  • make sure every line of code is executed at least once by test data

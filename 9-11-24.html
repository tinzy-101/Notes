Dynamically allocated memory cont'd

  • Pass a pointer in by reference when the memory location that the pointer points to needs to be changes
    • double*& p
    • passing in by value won't modify the pointer in main()

Bag Class with Dynamic Array

  1. for array, index from 1st element (0) to the last element + 1 (e.g. size 5, 5+1=6)
  2. copy constructor: pass in const ref object (to another bag)
    • if you write bag b = a, and you don't have a copy constructor, both b and a will be pointing to the same array
    • huge problem! need to create a new bag, and only copy constructor will do that
    • if class has pointer data member, MUST overload assignment operator, write destructor, and copy constructor
      • in our bag sequence, the data[] array is a pointer!
  3. passing in arguments as const ref is efficient
    • dont need to call copy onstructor OR destructor
  4. call destructor when function returns
  5. overloaded = needs to ignore statements like a = a
    • if(this == &source)
    • return
  6. when returning a new bag object, need to call copy constructor for temp variable

Const Notes

  1. int const** OR const int**
    • pointer to constant integer
    • the integer that is being pointed to should not be changed
    • where the pointer points can be changed
  2. int **const
    • constant pointer to an integer
    • integer value can change
    • where pointer points to cannot be changed
  3. const int** const OR int const** const
    • constant pointer pointing to constant integer
    • value stored in integer shouldnt change
    • where pointer points to should not be changed
  4. how to remember
    • start from the name of var and move clockwise to next pointer or type
    • start from right and move left
      • e.g. int <-- const <-- ** <-- const
        • const pointer (**) to const int
      • e.g. int *** const
        • const pointer to pointer to int
      • e.g. int ** const **
        • pointer to const pointer to int
      • e.g. int const ***
        • pointer to pointer to const int
      • e.g. int ** const ** const
        • const pointer to pointer pointer to int
    • pointer is const when ** is BEFORE the const (attached to start of const)
    • int/float/etc is const when ** is AFTER the const (attached to end of const) OR attached to END of int/float/etc

Runtime vs Compile Error

  1. compile time error:
    • usually semantics or syntax error
      • e.g. operations w/ different types, int + string
      • missing parens, semicolons
      • printing value of var w/o initializing
  2. runtime error:
    • usually logical, e.g. divide by 0, accessing var w/o initializing it

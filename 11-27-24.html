Unweighted Graphs cont'd

  1. representing vertices: vector of vector of
  2. representing edges: vector of vector of pointers to edge objects
  3. BFS: breadth first search
    • can choose any vertex as starting point, go level by level to reach every vertex
    • first root of resulting search tree, then all children of root, then all grandchildren of root from left to right
  4. degree (of vertex) = number of edges incident on a vertex
  5. BFS ON FINAL
    • tell contents of parents parray, visited bool array, search order
    • will need source code on the test to know order in which nodes are traversed (starting root node, which order to visit your children in)
      • on final: will always visit children smallest to largest (index: 0 -> 1 -2 -> etc)
  6. applications of BFS:
    • if num vertices searched = num vertices in graph, graph = connected
    • can find a cycle in the graph
    • finding shortest path between 2 vertices
